The one good trick

Weather is one HTTP GET and some regex

No account, no key, no telemetry. The skin asks Open-Meteo for the fields it wants, in the order it wants them, so the regex below can pull each value out by position. The numeric weather_code goes through a small Lua table that turns it into words like “Partly cloudy.”

[1]
Fetch
One GET to /v1/forecast with the fields listed in the URL.
[2]
Match
A single regex captures temp, code, wind, day/night by group.
[3]
Index
Each child measure reads one capture group — no extra calls.
[4]
Map
Lua turns the code number into a readable condition.
; Weather/Weather.ini  -> the whole widget, minus styling
[MeasureWeather]
Measure=WebParser
URL=https://api.open-meteo.com/v1/forecast?latitude=#Lat#&longitude=#Lon#
    ¤t=temperature_2m,weather_code,wind_speed_10m,is_day&timezone=auto
RegExp=(?siU)"temperature_2m":(.*),"weather_code":(.*),"wind_speed_10m":(.*),"is_day":(.*)}

[MeasureTemp]
URL=[MeasureWeather]
StringIndex=1    ; capture group 1 = temperature

[MeasureCode]
URL=[MeasureWeather]
StringIndex=2    ; -> weathercode.lua maps this to "Partly cloudy"
Structure

Four skins, one shared brain

Everything you'd change lives in a single Variables.inc — location, units, fonts, colors, which image to show. Swap your city and your character art and it's yours.

rainmeter-waifu-desktop/
├─ Waifu/ — character cutout (load several for more)
├─ Weather/ — live Open-Meteo, no key
├─ Clock/ — time + date
├─ Calendar/ — month grid, today highlighted (pure Lua)
├─ CpuTemp/ — hottest core via Core Temp plugin
└─ @Resources/ — Variables.inc · weathercode.lua · calendar.lua · Images/
